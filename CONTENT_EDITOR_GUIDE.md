# Content Editor Guide

This guide is for the person responsible for updating website content in Sanity Studio for the Knights of Columbus Corpus Christi Council 6188 website.

## What You Are Responsible For

You can safely manage:

- Homepage content
- Program pages: Faith, Family, Community, Life
- Program posts
- Events
- Officers

You are **not** responsible for:

- Website code changes
- Sanity schema changes
- Vercel deployments
- GitHub or repository management
- Environment variables or project settings

If something looks broken in the editor or on the website, contact the technical owner.

## Where You Work

Use the deployed **Sanity Studio** URL provided by the site owner.

Do not edit content in the codebase or local files unless you were specifically trained to do that.

## Logging In

1. Open the Sanity Studio URL
2. Sign in with your own Sanity account
3. Use only your assigned editor/content role
4. Do not share login credentials with others

## Content Types You Will Use

### Homepage

Use this to update:

- Hero title
- Hero subtitle
- Hero text
- Mission title
- Mission text
- Homepage images

### Programs

Use this to update the main content for:

- Faith
- Family
- Community
- Life

Each program page can have:

- Title
- Description
- Main content
- Featured image

### Program Posts

Use this to add news or updates for a specific program.

Each program post should include:

- Title
- Slug
- Program reference
- Date
- Excerpt
- Optional image
- Body content

**Important:** Always link the post to the correct program. If the program is wrong or missing, the post will not appear on the correct page.

### Events

Use this to add upcoming events.

Each event should include:

- Title
- Date
- Description
- Location
- Program reference

**Important:** Always select the correct program. Program pages only show events linked to that specific program.

### Officers

Use this to manage officer profiles.

Each officer can include:

- Name
- Position
- Bio
- Email
- Phone
- Photo
- Display order

## Recommended Publishing Workflow

1. Open the correct document
2. Make your content updates
3. Review for spelling, grammar, and image quality
4. Confirm the correct program is selected for posts/events
5. Publish the document
6. Check the live website after publishing

## Before You Publish

Use this checklist:

- Title is correct
- Dates are correct
- Images are uploaded and display properly
- Program reference is selected where required
- No unfinished draft text remains
- Links and descriptions look clean and complete

## After You Publish

Check the live website page that should show the update:

- Homepage changes should appear on the home page
- Program changes should appear on the matching program page
- Program posts should appear under the correct program page
- Events should appear on the matching program page and events page
- Officer changes should appear on the officers page

## If Changes Do Not Appear

Try these steps first:

1. Confirm the document was **published**, not just saved as a draft
2. Refresh the browser
3. Check the correct website page
4. Wait a minute and refresh again

If it still does not appear, contact the technical owner. In some cases, the website may need a redeploy.

## Best Practices

- Use your own account
- Keep titles short and clear
- Use consistent naming for posts and events
- Upload good-quality images
- Avoid publishing incomplete drafts
- Double-check program assignments before publishing
- Ask before deleting major content

## Contact the Technical Owner If

- A field is missing in Sanity Studio
- A document type is missing
- You cannot publish content
- The wrong content is showing on the website
- A program post or event does not appear where expected
- You need a new content type or layout

## Quick Summary

For normal content work, your job is simple:

- Edit content in Sanity Studio
- Link posts and events to the correct program
- Publish changes
- Check the live site
- Report any technical issues to the site owner
